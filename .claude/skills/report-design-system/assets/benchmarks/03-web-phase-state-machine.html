transport · 上行命令 · queue · 外部

PHASE STANDBY · standby=true · overlay=null
Standby() 全屏歡迎
machine._emit（l1）
歡迎光臨／輕觸螢幕；機器人沒 emit 過時 /api/state 回 _STANDBY 預設。
PHASE ORDERING · standby=false · overlay=null
Menu + CartRail
dialog io.display（每輪 + cart）
點餐主畫面；cart 空 / 非空 → empty / filled。後端 ordering ↔ 前端「無 overlay」。
PHASE CHECKOUT_CONFIRM · overlay="confirm"
ConfirmSheet() 確認卡
dialog io.display（_dialog_checkout_confirm）
明細 + 確認結帳／返回購物車。standby=false。
PHASE CHECKOUT · overlay="checkout"
CheckoutSheet() 結帳卡
machine._emit（l4）
QR + 我已完成付款。standby=false。
PHASE THANKYOU · overlay="thankyou"
ThankYou() 全屏謝謝惠顧
machine._emit（l5 · 帶 paid）
致謝；paid 僅此 phase 帶（falsy 時前端保留上一筆 paidTotal）。

機台 / dialog 轉態

所以呢 · phase 驅動三鐵則

  • ●phase 是唯一權威：畫面切換只認後端 emit 的 phase，禁前端樂觀改畫面。
  • ●三觸發殊途同歸：checkout_confirm 由語音 keyword / 沉默自動結帳 / UI 觸控三條路經同一 io.display ── UI 非唯一源。
  • ●斷線降級不卡死：WS onclose → resetToWelcome（回 standby）+「重新連線中」+ 指數退避（1s×2 上限 10s）。
