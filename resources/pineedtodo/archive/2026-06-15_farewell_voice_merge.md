## Step 1：重跑 TTS 預熱（產生合併句的預錄 mp3）

合併句是新的內容定址快取 key（`sha1(VOICE|rate|text)`）→ 無現成預錄資產。依文案常數改動 SOP：

**Pi 端**（需網路，**勿與 demo 同時跑**——同句 `.tmp` 會互踩寫壞檔）：
```bash
cd /home/pi/Desktop/project_jiqiren
git pull                              # 先同步本次 commit
python3.11 -m myProgram.tts_prewarm   # 自動枚舉，新合併句會被合成
```
預期輸出含 `[預熱] ✓ 付款成功，謝謝光臨，歡迎再來`，結尾 `[預熱] 完成：N 句新合成 / M 句已存在 / 0 句失敗`（舊「付款成功」「謝謝光臨，歡迎再來」兩句已存在會 skip，成為無害孤兒）。

**dev 端**（Pi 無 GitHub 認證、不從 Pi push）：scp 把新 mp3 拉回主機 → `git add myProgram/tts_cache` → commit。
> 回報我「預熱成功、新合成 1 句」，我可幫你把 `tts_cache` 拉回 commit；或你自行 scp + commit。

> 不跑預熱也能動：demo 線上時 edge-tts 會現場合成合併句、之後自我入快取；但 **斷網 demo 會缺這句聲音**。

## Step 2：複測結帳收尾

點一單 → 結帳 → 終端 `s`（或客服模式內 `scan`）模擬掃碼成功，確認：
- 聽到**單句**「付款成功，謝謝光臨，歡迎再來」（一句連貫；**非**分兩段、**非**「謝謝光臨歡迎再來」講兩次）。
- 動作 **bow → wave_hand**（鞠躬後揮手）順序不變。
- 之後 3s 靜默（致謝期）→ 回叫賣。
- 確認**無「付款成功」尾巴被截**（合併後 L4→L5 語音邊界消失，本應順帶免疫 ALSA drain 尾截）。

## 驗證
- 終端 `s` 結帳成功 → 一句合併語音 + bow→wave_hand + 3s 靜默回叫賣 = 合併成功。

**回報**：(1) 預熱是否成功（新合成 1 句）；(2) 複測單句語音 + 動作是否如期、有無尾巴截斷或「致謝講兩次」。
