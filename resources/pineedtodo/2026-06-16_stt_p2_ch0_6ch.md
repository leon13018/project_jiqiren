- **簡介**：STT 改讀 ReSpeaker **ch0（處理後：波束成形+降噪）**——`arecord` 由 `-c 1` 改 `-c 6` 再程式內反交錯取 ch0。需 `STT_ARECORD_DEVICE` 指向**原生 6 聲道**裝置，否則 `-c 6` 失敗。**無新 pip/apt 依賴。**

## Step 1 — 設環境變數（指向原生 6ch ReSpeaker，名稱版不隨 card 編號漂移）
```bash
echo 'export STT_ARECORD_DEVICE="hw:CARD=ArrayUAC10"' >> ~/.bashrc
source ~/.bashrc
```
（`ArrayUAC10` = `cat /proc/asound/cards` 裡 ReSpeaker 的 ID；若不同照實填。）

## Step 2 — 確認 6-channel 韌體
```bash
arecord -D hw:CARD=ArrayUAC10 --dump-hw-params -d 1 /dev/null 2>&1 | grep -i channels
```
應含 `CHANNELS: 6`（2026-06-16 已確認是 6ch 韌體；若顯示 1 需刷 6_channels_firmware.bin，回報我）。
