# L4：印金額 + 等掃碼（結帳層）

> **角色：顧客。** 機器人結算 cart 總額，等顧客掃碼付款。此層**有交易狀態**（cart 內已有商品），客服流程也跟其他層不同。

---

## 入口（從何處進來）

| 來源 | 條件 |
|---|---|
| L3 鏈路 C-1 | 顧客主動表達結帳意圖 |
| L3 鏈路 C-2 第二段 | 兩段 6s 都無回應 → 自動進結帳 |
| L4 客服模式「繼續交易」 | 顧客在客服模式選「繼續」後回 L4 主循環 |

---

## 進入時動作

- **計算總額**：依 [L0 商品列表](L0_共通.md#-商品列表) 對 cart 內每項商品算金額（含九折）
- **[終端]** 印金額明細：
  ```
  ====================================
  您的訂單：
    冰紅茶 ×2 = 54 元
    刮刮樂 ×1 = 180 元
  ------------------------------------
  總金額：234 元
  請掃碼付款（終端輸入 s + Enter 模擬掃碼成功）
  ====================================
  ```
- **[語音]** `"您的總金額是 234 元，請掃碼"`
- 初始化 `loop_count = 0`
- 初始化 `unclear_count = 0`
- 等待顧客回應 / 掃碼，最多 `WAIT_NO_RESPONSE` 秒（= 6s）

---

## 邏輯鏈路

### 鏈路 A — 掃碼成功 → L5

- **trigger**：終端輸入 `s` + Enter
- **進入動作**：[語音] TBD（S2 補；建議 `"付款成功"`）
- **轉到**：**L5**

### 鏈路 B — 拒絕（顧客主動取消）

- **trigger**：顧客回應命中「拒絕意圖」關鍵字（見 [L0 關鍵字白名單](L0_共通.md#-關鍵字白名單)）
- **進入動作**：
  - [語音] TBD（S2 補；建議 `"好的，取消這次交易，謝謝光臨"`）
  - **清空 `cart`**（整單作廢）
- **轉到**：L1 叫賣，**套用** [L0 子例程 A：回 L1 叫賣](L0_共通.md#子例程-a回-l1-叫賣)

### 鏈路 C — 想找客服（**特殊返回語義 — 與 L1/L2/L3 不同**）

- **trigger**：顧客回應命中「想找客服」關鍵字
- **進入動作**：
  - [終端] 印 [L0 客服電話](L0_共通.md#-客服電話placeholder)
  - **進入客服特殊模式**：跟 L1/L2/L3 客服「印完電話自動返回」不同，**留在客服模式內等待使用者明確決定**
  - [終端 + 語音] 提示兩個選項：`"請選擇『退出交易』或『繼續交易』（語音說或終端輸入 1=退出 / 2=繼續 皆可）"`
- **等待使用者選擇**（60s 客服模式 timeout — 60s 完全靜默 → 自動退出 + 清空 `cart` + 套子例程 A 回 L1，避免顧客拿到電話就走人造成系統卡死）：
  - **退出 trigger**（任一）：
    - 終端輸入 `1` + Enter
    - 語音 / 終端命中「退出交易意圖」關鍵字
    - 語音 / 終端命中「拒絕意圖」關鍵字（fallback — 顧客講「不買了」「不要了」等於要退出，無需強制顧客學「退出」一詞）
    - → 清空 `cart` → L1 叫賣（套子例程）
  - **繼續 trigger**（兩者任一）：
    - 終端輸入 `2` + Enter
    - 語音 / 終端命中「繼續交易意圖」關鍵字
    - → **回 L4 主循環狀態**（cart 保留 / `loop_count` 重置為 0 / 繼續等掃碼）
  - **掃碼 trigger**：
    - 終端輸入 `s` + Enter
    - → 視為「繼續交易」+ 立即觸發**鏈路 A 掃碼成功**（直接 → L5）
    - 設計理由：顧客在客服模式內按 `s` 的真實意圖 100% 是「我打完電話、要付款了」
  - **既不命中退出也不命中繼續** → 重複提示「請選擇：『退出交易』或『繼續交易』」並繼續等
  - **timeout trigger**：60s 內無任何回應（無語音、無終端輸入）
    - → 清空 `cart` → L1 叫賣，**套用**[L0 子例程 A：回 L1 叫賣](L0_共通.md#子例程-a回-l1-叫賣)

> **釐清點 #4 已敲定**：原 L4.md 寫「直到有用戶**手動終端輸入退出，或者語音告知機器人退出**」 — **兩者皆可**。我把「繼續」也設計成兩者皆可（保持互動對稱）。
>
> **為何 L4 客服跟 L1/L2/L3 不同**：L4 已進入「結帳中」狀態（cart 有商品 / 顧客準備付款），客服只是查詢電話、不一定要中斷交易；所以給使用者明確選擇權「要不要繼續」。L1/L2/L3 客服只是「印電話即返回」因為沒進行中的交易。

### 鏈路 D — 循環印金額（6s 沒任何回應 / 沒掃碼）

- **trigger**：等待 `WAIT_NO_RESPONSE`（= 6s）內顧客**沒有任何回應**（含沒掃碼）
- **進入動作**：
  - `loop_count += 1`
  - 印金額 + [語音]，內容依 `loop_count` 決定：

    | `loop_count` | 語音內容 | 語氣 |
    |---|---|---|
    | 1 | `"您的總金額是 X 元，請掃碼，或聯繫客服"` | 中性 |
    | 2 | `"提醒您，您的總金額是 X 元，需要協助請說『聯繫客服』"` | 柔提醒 |
    | 3, 4 | `"您的總金額是 X 元，請儘快完成掃碼"` | 中度催促（去掉「否則取消」威脅）|
    | 5, 6 | `"您的總金額是 X 元，請儘快完成掃碼，否則將取消這次交易"` | 明確警告 |

- **分流**：
  - `loop_count < L4_MAX_LOOPS`（= 6）→ 回到等待掃碼狀態
  - `loop_count == L4_MAX_LOOPS` 且該次循環 6s 後仍無回應 → **強制退**：
    - [語音] TBD（建議 `"已取消這次交易"`）
    - 清空 `cart`
    - L1 叫賣（套子例程）

> **釐清點 #3 更新敲定**：「循環 6 次」語義 = `loop_count` 最大值 = 6。從 L4 入口算起的時序：
> - `t=0`：入口印金額（**不算入 loop_count**）
> - `t=6`：第 1 次循環（loop_count=1）
> - `t=12`：第 2 次循環（loop_count=2）
> - `t=18`：第 3 次循環（loop_count=3）
> - `t=24`：第 4 次循環（loop_count=4）
> - `t=30`：第 5 次循環（loop_count=5）
> - `t=36`：第 6 次循環（loop_count=6）
> - `t=42`：第 6 次循環滿 6s 仍無回應 → 強制退
>
> **L4 等待總時長上限：42 秒**（不含中途客服模式可一直等的時間）。

### 鏈路 E — 無法判斷

- **trigger**：以下任一情況：
  - (a) 顧客回應**不命中** [L0 關鍵字白名單](L0_共通.md#-關鍵字白名單) 任一類別（且不是 `s` 終端輸入）
  - (b) 命中「想一下意圖」/「結帳意圖」/「商品」類關鍵字（這三類在 L4 不適用，視為無法判斷）
- **進入動作**：
  - `unclear_count += 1`
  - **分流**：
    - `unclear_count < 3` → **[語音]** `"不好意思我聽不太懂，如果想付款請掃碼，需要協助請說『聯繫客服』"` → 保持在本層回到等待狀態
    - `unclear_count == 3` → **[語音]** `"我可能無法協助您，正在為您聯繫客服"` → **自動進入鏈路 C 客服模式**（印電話 + 等選擇）
- **`unclear_count` 重置時機**：鏈路 A（掃碼成功）/ 鏈路 B（拒絕）觸發時重置為 0
- **不算入 `loop_count`**（兩個計數獨立）

## 判定優先序

**當顧客有回應時（依序檢查、先命中先返回）：**

1. 終端輸入 `s` → 鏈路 A（掃碼成功）
2. 拒絕意圖 → 鏈路 B
3. ~~想一下意圖~~（不適用 L4 — 已結帳、不再加單；若命中視為「無法判斷」走鏈路 E）
4. ~~結帳意圖~~（不適用 L4 — 已在結帳中）
5. 想找客服 → 鏈路 C（客服特殊模式）
6. ~~商品命中~~（不適用 L4 — 已結算總額；若命中視為「無法判斷」走鏈路 E）
7. 都沒命中 → 鏈路 E（無法判斷）

**當顧客 6s 內無任何回應：**
- → 鏈路 D（循環印金額 + loop_count++）

**當在客服特殊模式內等待選擇時：**
- 用「退出 / 繼續交易」白名單 + 終端 `1`/`2`，其他話重複提示。
