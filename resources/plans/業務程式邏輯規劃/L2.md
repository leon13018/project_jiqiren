## 邏輯鏈路

### 鏈路 A — 拒絕

- **trigger**（兩者任一）：
  - 等待超過 `WAIT_NO_RESPONSE`（6s）無任何回應
  - 顧客回應命中「拒絕意圖」關鍵字（見 [L0 關鍵字白名單](L0_共通.md#-關鍵字白名單)）
- **進入動作**：[語音] `"謝謝光臨"`（S2 接 TTS 前 stub 階段用 print 模擬即可）
- **轉到**：L1 叫賣，**套用** [L0 子例程 A：回 L1 叫賣](L0_共通.md#子例程-a回-l1-叫賣)

### 鏈路 B — 循環詢問需求

此鏈路有兩種子情況，兩種都讓顧客留在 L2 繼續對話。

#### B-1：無法判斷（顧客說的話聽不懂）

- **trigger**：顧客回應**不命中** [L0 關鍵字白名單](L0_共通.md#-關鍵字白名單) 任一類別
- **進入動作**：
  - **[語音]** `"不好意思我聽不太懂，請問要買什麼呢？或者您想聯繫客服？"`
- **保持在本層**：回到「進入時動作」的等待狀態（最多 `WAIT_NO_RESPONSE` 秒）

#### B-2：想找客服

- **trigger**：顧客回應命中「想找客服」關鍵字
- **進入動作**：[終端] 印 [L0 客服電話](L0_共通.md#-客服電話placeholder)
- **轉到**：印完**自動回 L2 循環詢問狀態**（無等待、無確認）

> **客服模式從 L2 進入的返回語義**：跟 L1 客服一樣**自動返回**。因為 L2 還沒進入「進行中的交易」，沒狀態要保留。對比 L4 客服（需手動「退出 / 繼續」）。

#### B-3：想一下（顧客猶豫）

- **trigger**：顧客回應命中「想一下意圖」關鍵字（`等等` / `等一下` / `稍等` / `想想` / `考慮` / `想一下` / `hold on` / `wait`，見 [L0 關鍵字白名單](L0_共通.md#-關鍵字白名單)）
- **進入動作**：
  - `think_count += 1`
  - **若 `think_count == 3`（第 3 次說「等等」）**：
    - 跳過 6s 沉默
    - **[語音]** TBD（S2 補；建議 `"看來您還在猶豫，謝謝光臨歡迎下次再來"`）
    - 直接走**鏈路 A 拒絕**邏輯（L2 cart 未建立、無需清空 → 套子例程 A 回 L1）
  - **否則（`think_count < 3`）**：
    - 機器人**沉默等待** `WAIT_NO_RESPONSE`（= 6s）給顧客思考時間
    - **分流**：
      - 6s 內顧客**有任何回應** → 立即處理該回應（重跑 [判定優先序](#判定優先序)）
      - 6s **仍無回應** → **[語音]** `"請問需要購買什麼東西嗎？"` → 回到 L2 主等待循環
- **`think_count` 重置時機**：鏈路 C（點到商品）/ 鏈路 A（拒絕，含 think_count==3 觸發的拒絕）觸發時重置為 0

> **與 L3 鏈路 B-4 的對稱性**：L2 / L3 都支援「想一下」鏈路，邏輯一致；都有 `think_count` 上限 2 次。差別：(1) 第 3 次走向不同 — L2 第 3 次走鏈路 A 拒絕（cart 未建、回 L1）；L3 第 3 次走鏈路 C-2 第二段（cart 有商品、推進結帳）。(2) 6s 後重問措辭不同 — L2 用「請問需要購買什麼東西嗎？」（未加單），L3 用「請問還需要什麼東西嗎？」（已加單）。

### 鏈路 C — 點到商品 → 進 L3

- **trigger**：顧客回應命中「商品 — 冰紅茶」OR「商品 — 刮刮樂」關鍵字
- **進入動作**：
  - **解析數量**：依 [L0 數量解析](L0_共通.md#-數量解析) 從顧客輸入抽取數量（例：「冰紅茶兩個」= 2）
  - **若顧客回應無數量（2026-05-25 加）** → **[語音]** `"請問您要幾{單位}？"`（模板 `QTY_PROMPT_TEMPLATE`，冰紅茶 → 瓶；刮刮樂 → 張）→ 等待顧客 follow-up `WAIT_NO_RESPONSE`（6s）→ 用 follow-up 數量；timeout 預設 1
  - 將命中商品 + 數量加入訂單（程式內 `cart` 結構，typically `cart.append((商品, qty))` 或 dict 合併同商品）
  - [語音] TBD（S2 補；建議 `"好的，已加入購物車"`）
- **轉到**：**L3**

## 判定優先序

依 [L0 關鍵字白名單判定邏輯](L0_共通.md#-關鍵字白名單)，但 L2 **跳過「結帳意圖」**（L2 還沒進入加單環節，「結帳」此時意義不明）：

1. 6s 超時 → 鏈路 A
2. 拒絕意圖 → 鏈路 A
3. **想一下意圖** → **鏈路 B-3**
4. ~~結帳意圖~~（不適用 L2）
5. 想找客服 → 鏈路 B-2
6. 商品命中 → 鏈路 C
7. 都沒命中 → 鏈路 B-1（無法判斷）

> 鏈路 B-3 思考期內顧客有回應 → 該回應**重跑 1-7 順序**。
