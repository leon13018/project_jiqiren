# L2：詢問需求（顧客互動 — 初次點單）

> **角色：顧客。** 顧客被 OpenCV 偵測到走近，機器人主動詢問需求。

---

| 來源 | 條件 |
|---|---|
| L1 叫賣模式 | OpenCV 偵測到人進入相機範圍 |
| L2 客服模式 B-2 | 印完電話自動返回此循環 |
| L2 鏈路 B-1 無法判斷 / B-3 想一下 6s 後重問 | 自動回到等待狀態 |

## 邏輯鏈路

### 鏈路 A — 拒絕

- **trigger**（兩者任一）：
  - 等待超過 `WAIT_NO_RESPONSE`（6s）無任何回應
  - 顧客回應命中「拒絕意圖」關鍵字（見 [L0 關鍵字白名單](L0_共通.md#-關鍵字白名單)）
- **進入動作**：[語音] `"謝謝光臨"`（S2 接 TTS 前 stub 階段用 print 模擬即可）
- **轉到**：L1 叫賣，**套用** [L0 子例程 A：回 L1 叫賣](L0_共通.md#子例程-a回-l1-叫賣)

### 鏈路 B — 循環詢問需求

此鏈路有兩種子情況，兩種都讓顧客留在 L2 繼續對話。

#### B-1：無法判斷（顧客說的話聽不懂）

- **trigger**：顧客回應**不命中** [L0 關鍵字白名單](L0_共通.md#-關鍵字白名單) 任一類別（含「結帳意圖」— L2 視為無法判斷）
- **進入動作**：
  - `unclear_count += 1`
  - **若 `unclear_count >= UNCLEAR_MAX`（= 3，2026-05-25 加）**：
    - **[語音]** `L2_UNCLEAR_REJECT_VOICE = "您似乎不確定要買什麼，謝謝光臨"`
    - 直接走**鏈路 A 拒絕**邏輯（L2 cart 未建立、無需清空 → 套子例程 A 回 L1）
  - **否則（`unclear_count < UNCLEAR_MAX`）**：
    - **[語音]** `"不好意思我聽不太懂，請問要買什麼呢？或者您想聯繫客服？"`
    - 保持在本層：回到「進入時動作」的等待狀態（最多 `WAIT_NO_RESPONSE` 秒）
- **`unclear_count` 重置時機**：任何被識別意圖（拒絕 / 想一下 / 客服 / 商品）觸發時重置為 0

#### B-2：想找客服

- **trigger**：顧客回應命中「想找客服」關鍵字
- **進入動作**：[終端] 印 [L0 客服電話](L0_共通.md#-客服電話placeholder)
- **轉到**：印完**自動回 L2 循環詢問狀態**（無等待、無確認）

> **客服模式從 L2 進入的返回語義**：跟 L1 客服一樣**自動返回**。因為 L2 還沒進入「進行中的交易」，沒狀態要保留。對比 L4 客服（需手動「退出 / 繼續」）。

#### B-3：想一下（顧客猶豫）

- **trigger**：顧客回應命中「想一下意圖」關鍵字（`等等` / `等一下` / `稍等` / `想想` / `考慮` / `想一下` / `hold on` / `wait`，見 [L0 關鍵字白名單](L0_共通.md#-關鍵字白名單)）
- **進入動作**：
  - `think_count += 1`
  - **若 `think_count == 3`（第 3 次說「等等」）**：
    - 跳過 6s 沉默
    - **[語音]** TBD（S2 補；建議 `"看來您還在猶豫，謝謝光臨歡迎下次再來"`）
    - 直接走**鏈路 A 拒絕**邏輯（L2 cart 未建立、無需清空 → 套子例程 A 回 L1）
  - **否則（`think_count < 3`）**：
    - 機器人**沉默等待** `WAIT_NO_RESPONSE`（= 6s）給顧客思考時間
    - **分流**：
      - 6s 內顧客**有任何回應** → 立即處理該回應（重跑 [判定優先序](#判定優先序)）
      - 6s **仍無回應** → **[語音]** `"請問需要購買什麼東西嗎？"` → 回到 L2 主等待循環
- **`think_count` 重置時機**：鏈路 C（點到商品）/ 鏈路 A（拒絕，含 think_count==3 觸發的拒絕）觸發時重置為 0

> **與 L3 鏈路 B-4 的對稱性**：L2 / L3 都支援「想一下」鏈路，邏輯一致；都有 `think_count` 上限 2 次。差別：(1) 第 3 次走向不同 — L2 第 3 次走鏈路 A 拒絕（cart 未建、回 L1）；L3 第 3 次走鏈路 C-2 第二段（cart 有商品、推進結帳）。(2) 6s 後重問措辭不同 — L2 用「請問需要購買什麼東西嗎？」（未加單），L3 用「請問還需要什麼東西嗎？」（已加單）。

### 鏈路 C — 點到商品 → 進 L3

- **trigger**：顧客回應**至少含一個商品 keyword**（紅茶 / 冰紅茶 / 刮刮樂 / 彩券 等；走 `parse_products` 多商品 parser，**2026-05-25 從單商品升級為多商品 B 方案**）
- **進入動作**：
  - **多商品解析**：`parse_products(response)` 回傳 `list of (product, qty_or_None)`，每商品的數量黏住該商品 keyword 後方視窗內第一個數字（sticky-right）
  - 範例：
    * 「紅茶 1 刮刮樂 2」→ `[(冰紅茶, 1), (刮刮樂, 2)]` → 一次全加
    * 「紅茶 1 刮刮樂」→ `[(冰紅茶, 1), (刮刮樂, None)]` → 紅茶直接加 1、刮刮樂進追問
    * 「紅茶 2 紅茶 3」→ `[(冰紅茶, 2), (冰紅茶, 3)]` → 累加 5 瓶（不視為誤說、不取覆蓋）
  - **對 list 內每個 (product, qty) pair：**
    * `qty` 不為 None → 直接加 cart
    * `qty` 為 None → **[語音]** `f"請問{product}要幾{單位}？"`（明示哪個商品）→ 進「QTY 追問 sub-loop」：
      - 含數量 → 用該數量；timeout → 預設 1；**客服** → 印電話 + clarify 繼續追問；**拒絕** → skip 該商品（其他仍加）；其他亂說 → speak clarify 繼續追問
  - **全部 pair 處理完**：若 cart 有至少一個新增 → speak `L2_C_ADDED` 進 L3；若全被取消 → speak `L2_B3_REASK` 回 L2 主迴圈
- **轉到**：**L3**（至少一個商品加入時）

## 判定優先序

依 [L0 關鍵字白名單判定邏輯](L0_共通.md#-關鍵字白名單)，但 L2 **跳過「結帳意圖」**（L2 還沒進入加單環節，「結帳」此時意義不明）：

1. 6s 超時 → 鏈路 A
2. 拒絕意圖 → 鏈路 A
3. **想一下意圖** → **鏈路 B-3**
4. ~~結帳意圖~~（不適用 L2）
5. 想找客服 → 鏈路 B-2
6. 商品命中 → 鏈路 C
7. 都沒命中 → 鏈路 B-1（無法判斷）

> 鏈路 B-3 思考期內顧客有回應 → 該回應**重跑 1-7 順序**。
