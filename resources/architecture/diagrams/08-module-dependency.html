所以呢 · 依賴鐵則

  • ●核心零向外依賴：sales/ 不 import 任何硬體 / vendor / web / edge_tts / websockets / fastapi；唯一外部 pypinyin 也是 lazy ＋ 注入 ＋ 缺則 no-op。
  • ●main.py 組合根唯一橋樑：top-level 只 import sales.logic（核心）；worker / web 全在 method 內 lazy import；callbacks() 10 鍵注入 logic.run。
  • ●依賴反轉：核心透過注入的 callback 函式指標 runtime 呼叫外部（speak / do_action / read_customer_input / display），原始碼層級對 adapter 零依賴 → 換 adapter 不動核心。
  • ●web / webui 與 sales 解耦：sales 不知 web 存在，只多呼叫注入的 display 回呼；web/ 的 bus · display · commands 亦純 stdlib → 核心與鏡像皆可單獨 pytest。
