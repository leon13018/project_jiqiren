並行模型要點

  • ●主線程純單線程、絕不阻塞:會阻塞 / 要並行的事全推背景 daemon。
  • ●三條 queue 各單 producer / 單 consumer;唯 input queue 例外 — 鍵盤 / STT / web 觸控三 producer 扇入同一 queue。
  • ●輸出走 queue(speak / do_action)與 EventBus(display);輸入全走 inject → input queue。
  • ●全部 daemon=True → 退出 os._exit(0) 強退,thread 隨 process die。
