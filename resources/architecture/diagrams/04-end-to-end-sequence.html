色彩語意 · 泳道角色

前端（render client） WS·transport
主線程（SalesMachine） STT（每輪）
TTS Action·vendor
phase emit 下行 display(phase) = 主角（金）

所以呢 · phase-driven 三鐵則

  • ●phase-driven：前端 7 次換畫面全由後端 display(phase) 驅動；觸控只送命令、不本地樂觀改畫面（觸發也可能來自語音 / 自動結帳）。
  • ●單一 input queue：wake/order/checkout/confirm/pay 觸控 token 與 STT speech_final 注入同一 queue，主線程 read_* 單一消費（呼應圖①）。
  • ●checkout_confirm 是 dialog 內子 phase：machine _PHASE_BY_STATE 只有 standby/ordering/checkout/thankyou 四個；確認卡片這發由 _dialog_checkout_confirm 直發 ★。
