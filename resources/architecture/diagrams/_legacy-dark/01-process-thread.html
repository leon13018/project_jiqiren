所以呢 · 並行模型三鐵則

  • ●全部 daemon=True → 主程式 os._exit(0) 強退，daemon 隨 process die，不 join。
  • ●阻塞 / 並行的事全推背景 worker；主線程只跑純單線程對話邏輯。
  • ●producer 端零分流：鍵盤 / 語音 / 觸控 → 同一 input queue，主線程單一消費（無旗號分流 race）。
