## 1. 背景與動機

`tts.py` / `action.py` 兩個 consumer worker 共用「FIFO queue + daemon thread + get→process 迴圈」骨架；drain-queue 迴圈在 tts / action / input_reader 重複 ×3；`main.py` 的倒數 polling loop 重複 ×2、finally shutdown 區塊重複 ×3、13 個 closure 散裝。W6 以 `QueueWorker(ABC)` 收 consumer 骨架、`TerminalSim` 類別收 main.py 的 callback 集（bound methods）。

**前置盤查（已完成）——凍結面與 seam**：
- `test_tts_worker.py`：`from myProgram.tts import TtsWorker`；**`TtsWorker()` 零參建構**；**`worker._pending` 屬性名直讀**（test 4f）；`say` / `wait_idle(max_wait=…)` 簽名；monkeypatch **`tts_module._synthesize`（模組屬性）** + `tts_module.subprocess.Popen` + `tts_module.time.sleep` → `_synthesize` 必須維持「處理函式內 module-global lookup」呼叫形式。

## 2. 設計核心 + 行為規約

### 2-1. QueueWorker 基底（新檔 `myProgram/queue_worker.py`）

```python
def drain_queue(q: "queue.Queue") -> None:
    """清空 queue（共用 helper——原 tts / action / input_reader 三份相同迴圈）。"""
    while not q.empty():
        try:
            q.get_nowait()
        except queue.Empty:
            break

class QueueWorker(ABC):
    """FIFO daemon 消費者骨架：queue + daemon thread + get → _process → on_done 迴圈。

    ⚠️ 子類別若有自有欄位（如 TtsWorker 的 _pending / _cv），必須在呼叫
    super().__init__() **之前**設好——基底 __init__ 立即啟動 daemon thread，
    thread 第一時間可能觸碰子類別欄位。
    """
    thread_name: str   # 子類別 class attr

    def __init__(self) -> None:
        self._q: "queue.Queue" = queue.Queue()
        threading.Thread(target=self._loop, name=self.thread_name, daemon=True).start()

    def submit(self, item) -> None:
        self._q.put(item)

    def _loop(self) -> None:
        self.on_thread_start()
        while True:
            item = self._q.get()
            try:
                self._process(item)
            finally:
                self.on_done(item)

    @abstractmethod
    def _process(self, item) -> None: ...
    def on_thread_start(self) -> None: pass   # ActionWorker：lazy import vendor
    def on_done(self, item) -> None: pass     # TtsWorker：dec _pending + notify
    def drain(self) -> None:
        drain_queue(self._q)
```

**與傘狀 §4-6 的兩處刻意收斂（傘狀隨本 wave 對齊）**：
1. **基底無 except-all / 無 on_error hook**——現行 TtsWorker `_loop` 只有 try/finally（未知例外殺 thread 是現狀）；ActionWorker 的 catch 在自己迴圈體內 → 改造後住進其 `_process`。基底加 catch-all 會改 TtsWorker 行為，不做。
2. **`shutdown` 不入基底**——兩 worker 順序相反（tts：terminate proc → drain；action：drain → 守衛 stopAction），各自保留現行實作、改用 `self.drain()` / `drain_queue` 取代手寫迴圈。

### 2-2. TtsWorker(QueueWorker)（`tts.py`）

- `thread_name = "TtsWorker"`；`__init__`：**先**設 `_proc` / `_lock` / `_cv` / `_pending = 0`，**後** `super().__init__()`（⚠️ §2-1 警告的具體案例）。
- `say(text)`：原樣保留（`with self._cv: self._pending += 1; self._q.put(text)`——原子性要求不經 base `submit`）。
- 原 `_loop` 刪除（骨架歸基底）；原 `_process_text(text)` 更名 `_process(text)`，**body 一字不改**（`asyncio.run(_synthesize(text, TMP_MP3))` 的 module-global lookup 保留——test seam）。
- `on_done(text)`：原 `_loop` finally body 搬入（`with self._cv: self._pending -= 1; if == 0: notify_all()`）。
- `wait_idle` 原樣；`shutdown`：terminate 段原樣，尾端清 queue 迴圈改 `self.drain()`。
- `_pending` / 模組級 `_worker` / `speak` / `speak_and_wait` / `wait_idle` / `shutdown` 對外 API 全部不動。

### 2-3. ActionWorker(QueueWorker)（`action.py`）

- `thread_name = "ActionWorker"`；`__init__`：只 `super().__init__()`。
- `on_thread_start()`：`from myProgram.vendor import ActionGroupControl as Act; self._act = Act`（= 原 `_loop` 頂端 lazy import，時機等價：thread 啟動後、首次 get 前；Windows import action 模組仍不觸發 vendor import）。
- `_process(name)`：原迴圈 body 搬入——`try: self._act.runAction(name) except Exception as e:` + **4 行 print 一字不改**。
- `do(name)`：`self.submit(name)`。
- `shutdown()`：`self.drain()` 取代手寫清空迴圈；守衛 stopAction 段（含自己的 lazy import）**原樣**。
- ⚠️ 無 Windows 測試 → sales-coder 逐行對照舊版自證；Pi 實機驗證兜底。

### 2-4. input_reader.py（不繼承）

- `InputReader` **不**繼承 QueueWorker（producer 形狀：daemon thread 是 readline→put 的生產者，`read()` 是主線程消費——與 consumer 骨架相反）。
- 僅 `shutdown()` 內手寫清空迴圈改 `drain_queue(self._q)`（import 自 queue_worker，純 stdlib 無循環）。
- `read()` 的 latest-wins drain **原樣不動**（語意不同：保留最新一筆）。

### 2-5. TerminalSim（`main.py`）

- **凍結**：`_S1State`（原樣不動）、`_build_callbacks(state)`（簽名不變，body 變 `return TerminalSim(state).callbacks()`）、`main()` 流程。
- `TerminalSim`：`__init__(self, state: _S1State)` 存 `self._state`；原 13 個 closure 逐一變方法（body 一字不改，`state.` → `self._state.`；**方法內 lazy import 原樣保留**——test seam）；`callbacks()` 回傳 dict（**14 鍵與現行完全一致**：print_terminal / read_terminal_key / opencv_dwell_seconds / opencv_disable / opencv_enable / mute_opencv / speak / speak_and_wait / do_action / read_customer_input / sleep / schedule / exit_program / show_hawk_help，值 = bound methods）。
- **倒數 helper 統一**（模組級函式）：

```python
def _tick_countdown(total: float, label: str, wait_tick):
    """每秒對齊整秒邊界倒數印 `{label} = N`；wait_tick(seconds) 回非 None 即中斷並回傳。"""
    deadline = time.monotonic() + total
    while True:
        remaining = deadline - time.monotonic()
        if remaining <= 0:
            return None
        ticks = math.ceil(remaining)
        print(f"{label} = {ticks}")
        got = wait_tick(remaining - (ticks - 1))
        if got is not None:
            return got
```

  - `read_customer_input`：`raw = _tick_countdown(timeout, "timeout", input_reader.read)`（read 回 None → 繼續倒數；回字串 → 中斷）——與現行迴圈逐分支等價；前後的 wait_idle / None-fallback / strip / normalize / q-exit 邏輯原樣。
  - `sleep`：`_tick_countdown(seconds, "wait", lambda s: time.sleep(s))`（lambda 恆回 None → 跑滿）——前置 wait_idle / None-or-≤0 fallback 原樣。`timeout = N` vs `wait = N` 的語意區分註解保留。
- **finally shutdown 收斂**：三段 try/except ImportError 改一個迴圈（`for name in ("tts", "action", "input_reader"): try: importlib.import_module(f"myProgram.{name}").shutdown() except ImportError: pass`）；各段的 why 註解濃縮為一段（cleanup 失敗不污染主流程 + Windows 無 edge_tts/vendor）；`os._exit(0)` 段原樣。
- 歷史演進敘事註解依 W1 政策處理（保留仍成立的 why、不揹日期演進史）。
