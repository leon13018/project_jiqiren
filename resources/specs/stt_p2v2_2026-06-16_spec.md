# STT Phase 2 v2 — 讀 ch0 + 來源端閘 prewarm（修正自我回授）spec

> 2026-06-16 定案。承接 `stt_p2_2026-06-16_spec.md`（Phase 2 v1，已 revert）。v1 的 prewarm 閘在**結果端**（照收機器人聲、丟棄轉錄）→ Deepgram 轉錄延遲讓機器人自聲漏過閘 → STT↔TTS 自我回授無限迴圈（Pi 實測，changelog 里程碑 6）。v2 把閘移到**來源端**根治。

## 1. 背景與動機

Phase 2 v1 為省「TTS 播完才連 Deepgram」的 ~300ms，做了 prewarm（播放期間就開麥、`_live` 閘丟棄辨識）。**Pi 實測自我回授**：機器人 TTS 在播放期被收音送 Deepgram，Deepgram「講完才回」的轉錄延遲使該結果在 `arm()` 開閘**之後**才到 → 機器人自聲被當顧客輸入 → 無限自我對話。**根因＝閘在結果端、擋不住延遲轉錄**（無 AEC 消不掉自聲）。v1 已 revert 回 Phase 1。

**v2 修正**：閘移到**來源端**——prewarm 期間**根本不送任何音訊進 Deepgram**（只開連線 + KeepAlive 維持），TTS 播完 `arm()` 才開始送音訊。Deepgram 從沒收到機器人聲 → 無轉錄 → 無回授。同時把 v1 的 ch0 讀取（與回授無關、被一起 revert）加回，改善「收音不好」。

## 2. 設計核心 + 行為規約

兩增量獨立、分開 commit。

### 增量 1 — 讀 ch0（重新加回，與 v1 increment 1 相同）
- `arecord` `-c 1`→`-c 6`（裝置須原生 6ch；Pi `STT_ARECORD_DEVICE=hw:CARD=ArrayUAC10`）。
- `_extract_ch0(buf)`：交錯 6ch S16 → 每幀取第 0 樣本（ch0 處理後）；不完整尾幀丟棄。
- `_ArecordSource.read(n)`：讀 `n*6` bytes → `_extract_ch0` → 回 mono ch0。

### 增量 2 v2 — 來源端閘 prewarm（核心修正）
- session 由 tuple 改 **dict**（`prewarm` 時建 ws+receiver+keepalive、`arm` 時才補 audio+sender）。
- 抽 `_open_ws()`（caller 持 lock）：連 ws + 起 receiver + 起 keepalive thread；缺 key/連線失敗 → 停用/放棄（同既有）。回傳是否成功。
- `prewarm()`：`_open_ws()`（**不起 arecord、不送音訊**）。冪等。
- `arm()`：`_open_ws()` 確保連線 → 若尚未送音訊則 `sending.set()`（停 keepalive）+ 起 arecord + sender。冪等。**單獨 arm（無 prewarm）= 即連即送，等同舊行為**（既有測試不破）。
- `_keepalive_loop(ws, stop, sending)`：每 `_keepalive_interval`（預設 5.0s、建構子可注入供測試）送 `{"type":"KeepAlive"}` text frame；`sending` set（已送音訊）或 `stop` 即止。防 Deepgram ~10s 無音訊斷線。
- `disarm()`：stop → close audio（若有）+ ws → join receiver/keepalive/sender。dict 版。
- **移除 `_live` sink 閘**——v2 不需要：音訊只在 arm 後送，Deepgram 只會回顧客辨識，receiver 照單注入即可。
- `main.py read_customer_input`：`prewarm()` → `wait_idle()` → `arm()`（wiring 同 v1）。

| 項目 | 規約 |
|---|---|
| prewarm 期送什麼 | **只送 KeepAlive**（text frame），**零音訊** → Deepgram 無轉錄 |
| arm 後 | 起 arecord 送 ch0 音訊；keepalive 停（audio 維持連線） |
| receiver 注入 | 無條件注入收到的 speech_final（只會是顧客聲；無 `_live` 閘） |
| 缺 key / 連線失敗 | prewarm/arm 皆 graceful no-op（純鍵盤照常） |
| TTS >10s 邊緣 | KeepAlive 兜住；極端兜不到 → 該輪 STT 失敗、下輪重連（不影響鍵盤） |
| disarm 冪等 | session=None 後 no-op；join 3 thread（receiver/keepalive/sender，None 跳過） |
