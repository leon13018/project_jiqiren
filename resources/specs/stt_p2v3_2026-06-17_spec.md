# STT Phase 2 v3 — 暖機期送靜音（arecord 提前、消除 arm 啟動延遲）spec

> 2026-06-17 定案。承接 v2（`stt_p2v2_2026-06-16_spec.md`，已上線、Pi 實測無自我回授）。v2 把 arecord 留在 `arm()` 才起 → 提示音播完後仍有 arecord 啟動延遲（使用者實測「播完不能馬上講、卡頓」）。v3 把 arecord 提前到 `prewarm`、暖機期送「靜音」取代真實音訊，`arm` 只解 mute → 消除啟動延遲。

## 1. 背景與動機

v2 流程：`prewarm`（連 ws + KeepAlive 維持、不起 arecord）→ `wait_idle` → `arm`（**才起 arecord** + 送音訊）。問題：arecord subprocess spawn + 首音框在 `arm`（提示音播完後）才發生 → 顧客感覺「播完到能講」有可感卡頓（使用者 2026-06-17 Pi 實測回報）。

v3（使用者構想）：**arecord 在 `prewarm` 就起、背景已在收音暖好**；暖機期 sender **讀到的真實聲音用「等長靜音」取代後送出**（機器人聲讀進來但永不送進 Deepgram，靜音同時維持連線、省掉 KeepAlive thread）；`arm` 只把旗號一翻 → sender 改送真實音訊。arecord 早暖好 → arm 後零啟動延遲，顧客一講即收。

**安全要點（v1 栽過）**：mute = 在**來源端**用靜音取代、**絕不送出機器人真實聲音**（非 v1「送出去再濾結果」）。`arm` 在 ALSA drain 之後（機器人已靜音）解 mute → 切換點乾淨。殘留 race 可忽略：解 mute 當下那一音框已是 post-drain 近靜音。

> 仍剩的 ~0.3s 是 ALSA drain（機器人尾音真的還在播，麥克風故意等它）——無解、不在本 spec 範圍。

## 2. 設計核心 + 行為規約

**單一增量**（ch0 已在 v2 內、不動；`main.py` prewarm→wait_idle→arm wiring 不變）。把 v2 的 keepalive-based prewarm 改為 silence-based：

- `__init__`：**移除** `keepalive_interval` 參數；session dict 改為 `stop/ws/receiver/audio/sender/live`。
- `_start_session(live_initial)`（取代 v2 `_open_ws`）：連 ws + 起 receiver + **起 arecord + sender**；起 sender 前依 `live_initial` 定 `live` 旗號（避免 mute 期誤送）。回傳是否就緒。
- `prewarm()`：`_start_session(live_initial=False)`（sender 進 mute、送靜音）。冪等。
- `arm()`：無 session → `_start_session(live_initial=True)`（未 prewarm 直接 arm＝即起即送，等同舊行為）；已 prewarm → `session["live"].set()`（解 mute）。冪等。
- `_send_loop(ws, audio, stop, live)`：每讀一 chunk → `ws.send(chunk if live.is_set() else b"\x00"*len(chunk))`。mute 期送等長靜音。
- **移除** `_keepalive_loop`（靜音已維持連線）。
- `disarm()`：dict 版，join `receiver/sender`（不再有 keepalive）。
- `_receive_loop` / `_connect_with_retry` 不動；**無 `_live` 結果端閘**（Deepgram 只收到靜音 + 顧客聲）。

| 場景 | 行為 |
|---|---|
| prewarm（提示音播放中） | arecord 已收音；sender 讀真實聲但**送等長靜音** → Deepgram 收到靜音、無轉錄 |
| arm（提示音播完，post-drain） | `live` set → sender 改送真實音訊；arecord 已暖 → 零啟動延遲 |
| 連線維持 | 靜音串流維持（無需 KeepAlive） |
| 缺 key / 連線失敗 | prewarm/arm graceful no-op（純鍵盤照常） |
| disarm | stop + close audio + ws + join receiver/sender；冪等 |
| 未 prewarm 直接 arm | `_start_session(live_initial=True)`：即起即送（向後相容，既有測試不破） |
