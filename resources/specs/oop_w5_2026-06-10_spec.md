# oop_w5 — W5 SalesMachine + State pattern（SDD spec）

> 傘狀設計：`resources/specs/oop_refactor_2026-06-10_design.md`（W5 列、§4-5）。
> 純重構：**對外行為零改變、既有測試零修改**。收掉 `logic.py` 主迴圈的 tuple 魔法值與 if/elif 調度。

## 1. 背景與動機

`logic.py` 主迴圈以字串 tuple（`("L4", 0)`、`("L1_via_subroutine_a", 0, 0)`）+ if/elif 鏈調度 L1→dialog→L4→L5 cycle。W5 以教科書 **State pattern** 表達：`State(ABC)` + `Transition` dataclass + `SalesMachine` 主迴圈；`logic.run` 變 facade。tuple 魔法值死在 machine 內部（各 `run_*` 對外回傳 shape 不變——它們被測試直接呼叫）。

**前置盤查（已完成，sales-coder 不必重驗）**：
- `test_logic.py` 的 mock seam = **`myProgram.sales.states` 模組屬性**（`monkeypatch.setattr(states_module, "run_l1", stub)` 等 5 個）→ State 子類別**必須晚綁定** `states.run_*(...)`（模組屬性呼叫），禁 `from ... import run_l1` 捕引用。
- 測試 stub 是**嚴格 keyword-only 簽名**（多/少一個 kwarg 都 TypeError）→ 呼叫 kwargs 逐字照抄現行 `logic.py` 各呼叫點。
- `run_l5` 回傳值被現行碼**無條件忽略**（test 5 stub 回 `("L1",0,0)` 也過）→ `L5State` 照樣忽略。
- `_assert_cart_empty` / `_assert_cart_nonempty` / `_invoke_subroutine_a` 僅測試 docstring 提及、無 import → 可搬移至 machine.py，不需 re-export。
- AssertionError 訊息被 `pytest.raises(match="Cart invariant 違反")` 釘住 → 訊息格式與 6 個 ctx 字串原樣保留。

## 2. 設計核心 + 行為規約

### 2-1. Transition + State（新檔 `myProgram/sales/states/machine.py`）

```python
@dataclass(frozen=True)
class Transition:
    """狀態轉移結果——取代主迴圈的字串 tuple 魔法值。"""
    next_state: str                 # "dialog" / "l4" / "l5" / "l1"
    via_subroutine_a: bool = False  # True = 先跑子例程 A + 下輪 L1 直接 hawk

class State(ABC):
    """機台狀態基底。entry_invariant + entry_ctx 供 machine 進場前 fail-fast 檢查（A4-c）。"""
    entry_invariant: str   # "empty" / "nonempty"
    entry_ctx: str         # assert 訊息用語境字串（原樣保留）

    @abstractmethod
    def run(self, machine) -> "Transition | None": ...   # None = 程式終止（run_l1 的 None）
```

（與傘狀 §4-5 的差異：`cart_invariant` 更名 `entry_invariant` + 增 `entry_ctx`；`run` 回傳放寬為 `Transition | None` 表達 L1 終止——傘狀 §4-5 隨本 wave 一併對齊。）

### 2-2. 四個 State 子類別（行為 = 現行 logic.py 逐段搬移）

| State | entry_invariant / ctx | run 內容（kwargs 逐字照抄現行呼叫點） | 轉移 |
|---|---|---|---|
| `L1State` | empty /「進 L1」 | `states.run_l1(print_terminal=…, read_terminal_key=…, opencv_dwell_seconds=…, opencv_disable=…, opencv_enable=…, speak=…, do_action=…, exit_program=…, schedule=…, show_hawk_help=…, enter_hawk_immediately=machine.enter_hawk_immediately)`；呼叫後**立即** `machine.enter_hawk_immediately = False`（consume-reset） | `"L2"` → `Transition("dialog")`；`None` → `None`（終止） |
| `DialogState` | empty /「進 dialog」 | `states.run_dialog(speak=…, print_terminal=…, read_customer_input=…, cart=machine.cart, think_count=0, opencv_disable=…, do_action=…, speak_and_wait=…)` | `("L4", _)` → `Transition("l4")`；`("L1_via_subroutine_a", _)` → 先 `_assert_cart_empty(cart, "dialog 退出後（dialog A 已視情況清 cart）")` → `Transition("l1", via_subroutine_a=True)` |
| `L4State` | nonempty /「進 L4」 | `states.run_l4(speak=…, print_terminal=…, read_customer_input=…, cart=…, opencv_disable=…, do_action=…, speak_and_wait=…)` | `("L5", _, _)` → `Transition("l5")`；其他 → 先 `_assert_cart_empty(cart, "L4 非掃碼退出後（L4 三條清 cart 路徑）")` → `Transition("l1", via_subroutine_a=True)` |
| `L5State` | nonempty /「進 L5（從 L4-A 帶 cart）」 | `states.run_l5(speak=…, cart=…, sleep=…, do_action=…)`；**回傳值忽略**（前置盤查） | 無條件：先 `_assert_cart_empty(cart, "L5 退出後（L5 應已清 cart）")` → `Transition("l1", via_subroutine_a=True)` |

- 現行 logic.py 各段的行為性註解（subroutine_a 出口涵蓋說明 / P3.C 修訂註 / B 方案層狀態判定原則段）搬到對應新位置。

### 2-3. SalesMachine

```python
class SalesMachine:
    """L1→dialog→L4→L5 主迴圈：持 cart + callbacks + enter_hawk 旗號；每次進狀態先驗 cart invariant。"""
    def __init__(self, callbacks: dict, cart):
        self.callbacks = callbacks          # logic.run 收到的 13 個 callback（含 speak_and_wait）
        self.cart = cart
        self.enter_hawk_immediately = False
        self._states = {"l1": L1State(), "dialog": DialogState(), "l4": L4State(), "l5": L5State()}

    def run(self) -> None:
        current = "l1"
        while True:
            state = self._states[current]
            if state.entry_invariant == "empty":
                _assert_cart_empty(self.cart, state.entry_ctx)
            else:
                _assert_cart_nonempty(self.cart, state.entry_ctx)
            result = state.run(self)
            if result is None:
                return None
            if result.via_subroutine_a:
                states.run_subroutine_a(mute_opencv=self.callbacks["mute_opencv"])   # 晚綁定
                self.enter_hawk_immediately = True
            current = result.next_state
```

- `_assert_cart_empty` / `_assert_cart_nonempty` 自 logic.py **原樣搬入** machine.py（訊息格式 `Cart invariant 違反 [{ctx}]：…` 一字不改）；`_invoke_subroutine_a` 內聯為上面一行（其簡化歷史註解搬到 machine docstring 或該行旁）。
- **等價性論證（invariant 檢查時點）**：現行 logic 在「進每層前」assert + 「L1_via_subroutine_a 返回後」assert。machine 把前者放主迴圈進場、後者放各 State 的轉移分支（§2-2）——時序與現行逐一對應：dialog/L4 退出 assert 在 sub_a 之前 ✓（現行亦然）、L5 後 assert 在 sub_a 之前 ✓。
- 單例不適用：`SalesMachine` 每次 `logic.run` 新建（cart 生命週期 = 一次 run）；State 子類別無狀態、表內實例可重用。
