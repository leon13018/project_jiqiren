> 傘狀設計：`resources/specs/oop_refactor_2026-06-10_design.md`（W2 列）。
> 純重構：**對外行為零改變、既有測試零修改**。收掉盤點 #2（`_speak_blocking` fallback ×8）與 #5（callback 參數鏈 7-8 參 × 20+ 呼叫點）。

## 1. 背景與動機

sales 對話側每個私有函式都手傳 5-8 個 callback 參數，每個呼叫點佔 7-9 行；`_speak_blocking = speak_and_wait if speak_and_wait is not None else speak` 重複 8 處。W2 引入 `DialogIO` callback 束物件，私有函式一律收 `io` 單參。

## 2. 設計核心 + 行為規約

### 2-1. DialogIO（新檔 `myProgram/sales/dialog_io.py`）

```python
@dataclass(frozen=True)
class DialogIO:
    """對話層 IO callback 束。只裝 IO，不裝業務狀態（cart / 計數器仍獨立傳）。

    print_terminal / do_action / speak_and_wait 預設 None：confirm 類子狀態只持有
    部分 callback（如 cancel_confirm 無 print_terminal），允許部分注入；
    缺欄位的子狀態不得使用該欄位。production 完整 wire-up 由 run_* facade 全欄注入。
    """
    speak: Callable
    read_customer_input: Callable
    print_terminal: Callable = None
    do_action: Callable = None
    speak_and_wait: Callable = None

    def speak_blocking(self, text: str) -> None:
        """阻塞 speak（wall-clock budget 用）——取代 8 處 fallback 三元式，語意一字不差。"""
        (self.speak_and_wait if self.speak_and_wait is not None else self.speak)(text)
```

- **與傘狀設計 §4-2 的差異**：`print_terminal` / `do_action` 加 `= None` 預設（部分注入需求，理由如上 docstring）；傘狀 §4-2 隨本 wave 一併修正。
- `frozen=True`：callback 束建好不重指派。
- `opencv_disable` **不入 io**：只在 `run_dialog` / `run_l4` facade 進場呼叫一次，私有函式不用。

### 2-2. 簽名規則（一條規則，reviewer 逐函式核）

### 2-3. facade 與凍結函式的內部改寫

- **facade 建 io**：`run_dialog` / `run_l4` 第一步 `io = DialogIO(speak=speak, read_customer_input=read_customer_input, print_terminal=print_terminal, do_action=do_action, speak_and_wait=speak_and_wait)`，之後 facade 體內與下傳一律用 `io`。
- **凍結簽名的 helper 內部也建 io**（殺自己的 `_speak_blocking` 行）：
  - `cancel_confirm` → `io = DialogIO(speak=speak, read_customer_input=read_customer_input, speak_and_wait=speak_and_wait)`
  - `service_confirm` / `invalid_qty_reask` / `invalid_qty_cancel_confirm` / `resolve_and_add_products` → 同 pattern（各帶自己持有的欄位）
- **跨模組呼叫凍結函式時自 io 展開**（W3 前的過渡，刻意為之）：如 `cancel_confirm(io.speak, io.read_customer_input, speak_and_wait=io.speak_and_wait)`、`service_confirm(speak=io.speak, print_terminal=io.print_terminal, ...)`。
- 8 處 `_speak_blocking = ...` 全數刪除，改 `io.speak_blocking(...)`。

### 2-4. 行為零改變確認點（reviewer 逐條核）

1. `io.speak_blocking` 語意 = 原三元式（`is not None` 判斷，非 truthiness）。
2. 所有 read timeout / deadline / budget / 計數器邏輯不動——只換參數載體。
3. `opencv_disable` 呼叫點（run_dialog / run_l4 進場各一）原位不動。
4. `print_terminal` 在 `l2_l3_dialog.py` 原本即純 pass-through（無直接呼叫點）——參數消失不影響行為。
5. 不變式 #4（wall-clock 從 speak_and_wait 播完起算）：`speak_blocking` 呼叫位置原位不動。
