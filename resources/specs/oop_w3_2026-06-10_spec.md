## 1. 背景與動機

`cancel_confirm`（6s）/ `service_confirm`（24s）/ `invalid_qty_cancel_confirm`（6s）三個函式共用同一 wall-clock 骨架，只差：提示文案、秒數、keyword 分類、timeout 預設值、亂答行為、進場動作。W3 以 **Template Method** 把骨架收成唯一一份，差異下放子類別覆寫點——這是本次 OOP 重構第一個真正的多型構件。同時內部呼叫點改用類別實例，消滅 W2 留下的「io 展開回 kwargs」過渡寫法與其過渡註解。

## 2. 設計核心 + 行為規約

### 2-1. TimedConfirm 基底（新檔 `myProgram/sales/states/_timed_confirm.py`）

```python
class TimedConfirm(ABC):
    """計時確認子狀態 Template Method。

    共用骨架：on_enter → speak_blocking(prompt) → wall-clock 迴圈
    （超時 / 沉默 → on_timeout；classify 命中 → return；亂答 → on_unclear，不重置 deadline）。
    """
    prompt: str        # 子類別 class attr
    timeout: float     # 子類別 class attr

    def run(self, io):
        self.on_enter(io)
        io.speak_blocking(self.prompt)
        deadline = time.monotonic() + self.timeout
        while True:
            remaining = deadline - time.monotonic()
            if remaining <= 0:
                return self.on_timeout()
            response = io.read_customer_input(timeout=remaining)
            if response is None:
                return self.on_timeout()
            outcome = self.classify(response)
            if outcome is not None:        # ⚠️ 必須 is not None——False 是合法結果（CancelConfirm NO）
                return outcome
            self.on_unclear(io)

    @abstractmethod
    def classify(self, response: str): ...   # keyword → 結果；None = 亂答
    @abstractmethod
    def on_timeout(self): ...                # 保守 default（不變式 #1）
    def on_enter(self, io): pass             # 預設 no-op
    def on_unclear(self, io): pass           # 預設 silent 消耗預算
```

### 2-2. 三個子類別（行為規約 = 現行三函式逐分支對齊）

| 子類別 | prompt / timeout | classify（**行序＝規格**） | on_timeout | on_unclear | on_enter |
|---|---|---|---|---|---|
| `CancelConfirm` | `CANCEL_CONFIRM_PROMPT` / `CANCEL_CONFIRM_TIMEOUT` | `KG_CANCEL_CONFIRM_NO`→`False`；`KG_CANCEL_CONFIRM_YES`→`True`；else `None` | `True`（取消） | silent（繼承預設） | 繼承 no-op |
| `ServiceConfirm(allow_scan)` | `L4_C_CONFIRM_PROMPT_TEMPLATE.format(seconds=L4_C_CONFIRM_TIMEOUT)` / `L4_C_CONFIRM_TIMEOUT` | ① `allow_scan and response == "s"`→`"scan"`；② `KG_L4_C_CONFIRM_NO`→`"no"`；③ `KG_L4_C_CONFIRM_YES`→`"yes"`；else `None` | `"no"` | `io.speak(L4_UNCLEAR_NOTICE)`（非阻塞） | `io.print_terminal(SERVICE_PHONE)` |
| `InvalidQtyCancelConfirm` | `INVALID_QTY_CANCEL_CONFIRM_PROMPT` / `INVALID_QTY_CANCEL_CONFIRM_TIMEOUT` | `KG_INVALID_QTY_CONTINUE`→`"cancel_overlimit"`；`KG_INVALID_QTY_EXIT`→`"exit"`；else `None` | `"cancel_overlimit"`（保 cart） | `io.speak_blocking(INVALID_QTY_UNCLEAR_PREFIX + self.prompt)` | 繼承 no-op |

- `ServiceConfirm` 帶 `__init__(self, allow_scan: bool = False)` 實例欄位；其餘子類別無實例狀態。
- **模組級單例**（皆無可變狀態，安全共用）：`CANCEL_CONFIRM = CancelConfirm()`、`SERVICE_CONFIRM = ServiceConfirm(allow_scan=False)`、`SERVICE_CONFIRM_SCAN = ServiceConfirm(allow_scan=True)`、`INVALID_QTY_CANCEL_CONFIRM = InvalidQtyCancelConfirm()`。
- import 鏈：`_timed_confirm.py` 只 import stdlib + `constants`（prompt / timeout / KG_*）→ 無循環。
