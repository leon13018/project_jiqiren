## 2. 設計核心 + 行為規約

### 2-1. ModePolicy（`l2_l3_dialog.py` 內部，ABC + 兩單例）

```python
class ModePolicy(ABC):
    """L2/L3 模式差異集中點（Strategy）。資料欄位 = 子類別 class attrs；行為差異 = hook。"""
    nlu_mode: str                  # classify_intent / resolve 的 mode
    read_timeout: float            # 主迴圈 read timeout
    entry_prompt: str
    clarify: str                   # B-1 文案
    reask: str                     # 沉默重問 / 商品全 skip 重問文案
    cancel_declined_resume: str    # cancel_confirm NO 後合成 voice
    think_limit: int
    service_yes_prompt: str        # service_confirm YES 後重啟文案
    silence_think_writeback: bool  # 沉默鏈 think 增量是否回寫主迴圈（quirk Q1）

    @abstractmethod
    def on_timeout(self, session) -> tuple: ...
    @abstractmethod
    def on_think_exhausted(self, session) -> tuple: ...
    @abstractmethod
    def on_checkout_main(self, session): ...     # 主迴圈語境（碰 unclear）
    @abstractmethod
    def on_checkout_inner(self, session): ...    # 沉默期語境（不碰 unclear；quirk Q2）
    @abstractmethod
    def on_unclear_exhausted(self, session): ...
```

| 欄位 / hook | `L2Policy`（cart 空） | `L3Policy`（cart 非空） |
|---|---|---|
| `nlu_mode` | `"l2"` | `"normal"` |
| `read_timeout` | `DNC_TIMEOUT` | `DYC_TIMEOUT` |
| `entry_prompt` | `L2_ENTRY_PROMPT` | `L3_ENTRY_PROMPT` |
| `clarify` | `L2_B1_CLARIFY` | `L3_B1_CLARIFY` |
| `reask` | `L2_B3_REASK` | `L3_REASK` |
| `cancel_declined_resume` | `L2_CANCEL_DECLINED_RESUME` | `L3_CANCEL_DECLINED_RESUME` |
| `think_limit` | `3` | `4` |
| `service_yes_prompt` | `L2_ENTRY_PROMPT` | `L3_REASK` |
| `silence_think_writeback` | `False` | `True` |
| `on_timeout` | speak `L2_TIMEOUT_TO_HAWK_VOICE`；return `("L1_via_subroutine_a", 0)` | return `session.c2_second_stage()` |
| `on_think_exhausted` | speak `L2_B3_THIRD_REJECT`；return `session.exit_a()` | return `session.c2_second_stage()` |
| `on_checkout_main` | `session.unclear_count += 1`；達 `UNCLEAR_MAX` → speak `L2_UNCLEAR_REJECT_VOICE` + return `session.exit_a()`；否則 speak `clarify` + return None | `session.unclear_count = 0`；return `session.checkout_flow()` |
| `on_checkout_inner` | speak `L2_B1_CLARIFY`；return None（**不動 unclear**） | return `session.checkout_flow()`（**不 reset unclear**） |
| `on_unclear_exhausted` | speak `L2_UNCLEAR_REJECT_VOICE`；return `session.exit_a()` | `_dialog_unclear_final_confirmation(io, cart)`；tuple → return 之；None → `session.unclear_count = 0`、`session.think_count = 0`、speak `L3_ENTRY_PROMPT`、return None |

單例：`L2_POLICY = L2Policy()`、`L3_POLICY = L3Policy()`（無可變狀態）。
（測試對模組常數 0 patch 已驗證 → class attr 於 import 時捕值安全。）

### 2-2. DialogSession（`l2_l3_dialog.py` 內部）

```python
class DialogSession:
    """持 io + cart + 計數器；對話主迴圈與沉默期分派。

    計數器歸 session 所有 → 消滅原 tuple|int|None 回傳型手動狀態穿線。
    """
    def __init__(self, io, cart, think_count: int = 0):
        self.io = io
        self.cart = cart
        self.think_count = think_count
        self.unclear_count = 0

    def policy(self) -> ModePolicy:
        # 每次呼叫由 cart 即時推導，不快取——「世界狀態驅動」核心決定（不變式 #8）
        return L2_POLICY if cart_module.is_empty(self.cart) else L3_POLICY
```

方法（取代原 5 個函式 + 4 個移為方法）：
- `main_loop() -> tuple`（取代 `_dialog_main_loop`；**入口先 `self.unclear_count = 0`**——保留「每次（重）入主迴圈 unclear 從 0 起算」語意，含 c2 CONTINUE 重入）
- `_think_silence() -> tuple | None`（取代 `_dialog_think_silence_l2/_l3` 雙胞胎）
- `_dispatch_inner(response) -> tuple | None`（取代 `_dialog_dispatch_inner_l2/_l3` 雙胞胎）
- `checkout_flow() -> tuple | None`（抽出主迴圈 L3 結帳 / inner L3 結帳共用體：confirm → "yes"→speak GO + do_action(GO) + `("L4", 0)`；"cancel_to_l1"→`exit_a()`；其他→`_handle_checkout_confirm_result` + None）
- `c2_second_stage() -> tuple`、`_c2_checkout_via_confirm() -> tuple`（原函式邏輯**逐行原樣**移為方法，引用改 self.*；C-2 三選一語意零改——不變式 #6）
- `exit_a() -> tuple`（原 `_dialog_exit_a`）
- `_reenter_speak(control)`（抽出三處重複的 reenter prefix + 當前 cart entry speak）

**維持模組函式不動**（簽名 `(io, cart)` 不變）：`_dialog_checkout_confirm`、`_dialog_unclear_final_confirmation`、`_handle_checkout_confirm_result(result, cart, io)`、`_prepend_cancel_notices`、`_build_order_summary`。

`run_dialog` facade（簽名凍結）：建 io → opencv_disable → entry do_action/speak（現行兩行三元保留）→ `return DialogSession(io, cart, think_count).main_loop()`。

### 2-3. main_loop 統一後流程（與現行逐分支對齊）

```
self.unclear_count = 0
while True:
    policy = self.policy()                      # 每輪推導
    response = io.read_customer_input(timeout=policy.read_timeout)
    if response is None: return policy.on_timeout(self)
    intent = classify_intent(response, policy.nlu_mode)
    拒絕:    CANCEL_CONFIRM.run(io) True → return self.exit_a()
             False → speak policy.cancel_declined_resume；continue
    想一下:  self.unclear_count = 0；self.think_count += 1
             if self.think_count >= policy.think_limit: return policy.on_think_exhausted(self)
             saved = self.think_count
             result = self._think_silence()
             if isinstance(result, tuple): return result
             if not policy.silence_think_writeback:           # Q1：L2 沉默鏈增量不回寫
                 self.think_count = saved
                 if not cart_module.is_empty(self.cart):      # B11：L2→L3 切換 reset
                     self.think_count = 0
             continue
    結帳:    result = policy.on_checkout_main(self)
             if isinstance(result, tuple): return result
             continue
    客服:    self.unclear_count = 0；SERVICE_CONFIRM.run(io)
             "yes" → speak policy.service_yes_prompt；continue
             "no"  → return self.exit_a()
    商品（parse_products 非空）:
             self.unclear_count = 0；was_empty = cart_module.is_empty(self.cart)
             resolve_and_add_products(..., classify_intent_mode=policy.nlu_mode, ...)
             control=="exit_l1" → return self.exit_a()
             control reenter_* → self._reenter_speak(control)；continue
             added: was_empty → self.think_count = 0（B11）+ io.do_action(ACTION_L3)
                                + speak prepend(L2_TO_L3_TRANSITION)
                    else → speak prepend(L3_REASK)
                    continue
             全 skip → speak prepend(policy.reask)；continue
    想買無商品: speak DIALOG_VAGUE_BUY_REASK；continue（不動任何計數）
    其他:    self.unclear_count += 1
             if self.unclear_count >= UNCLEAR_MAX:
                 result = policy.on_unclear_exhausted(self)
                 if isinstance(result, tuple): return result
                 continue
             speak policy.clarify
```

⚠️ 三處注意：(a) 商品 added/全 skip 分支的文案選擇基準與現行一致（added 用 `was_empty` 判 TRANSITION/L3_REASK；全 skip 用迭代頂端 cart 狀態的 reask——cart 未變時兩者等價，照現行寫法以 `was_empty` 對應之 policy 取 reask 亦可，sales-coder 以「行為等價」為準）；(b) 拒絕 / 想買無商品分支**不**碰 unclear（現行如此）；(c) `_reenter_speak` 的 entry 依 resolve **後**的當前 cart 即時判定（現行 `cart_module.is_empty(cart)` 重算）。

### 2-4. `_think_silence` / `_dispatch_inner` 統一後流程

```
_think_silence():
    inner = io.read_customer_input(timeout=WAIT_NO_RESPONSE)
    if inner is None: io.speak(self.policy().reask)；return None
    return self._dispatch_inner(inner)

_dispatch_inner(response):
    policy = self.policy()
    intent = classify_intent(response, policy.nlu_mode)
    拒絕:    gate True → return self.exit_a()；False → speak policy.cancel_declined_resume；return None
    想一下:  self.think_count += 1
             if self.think_count >= policy.think_limit: return policy.on_think_exhausted(self)
             return self._think_silence()                     # 互遞迴（原樣）
    結帳:    return policy.on_checkout_inner(self)            # Q2：inner 不碰 unclear
    客服:    SERVICE_CONFIRM.run(io)；"yes" → speak policy.service_yes_prompt；return None
             "no" → return self.exit_a()
    想買無商品: speak DIALOG_VAGUE_BUY_REASK；return None
    商品:    was_empty = is_empty(cart)；resolve(..., mode=policy.nlu_mode)
             exit_l1 → return self.exit_a()
             reenter_* → self._reenter_speak(control)；return None
             added 且 was_empty → io.do_action(ACTION_L3) + speak prepend(TRANSITION)；return None
             （Q3：inner 的 added 不 reset think——B11 由主迴圈處理，現行註解原樣保留）
             added 且非 was_empty → speak prepend(L3_REASK)；return None     ← 原 inner_l3 不分 added 一律 reask，等價
             全 skip → speak prepend(policy.reask)；return None
    其他:    speak policy.clarify；return None
```

**等價性論證（inner 商品分支）**：原 `inner_l2` 只在 cart 空語境被呼叫（added ⇒ was_empty 必真 → TRANSITION 分支 = 原行為）；原 `inner_l3` 只在 cart 非空語境（was_empty 必假 → 不論 added 一律 prepend reask = 原 `_added` 未用之行為）。統一版以 `was_empty` 分流在兩種語境下各自退化為原函式行為。

### 2-5. 行為差異矩陣（mode × 語境——reviewer 逐格核對的權威表）

| 事件 | 主迴圈 L2 | 主迴圈 L3 | inner L2 | inner L3 |
|---|---|---|---|---|
| read timeout | DNC | DYC | WAIT_NO_RESPONSE（silence 讀） | 同左 |
| timeout/None | hawk voice + 退 L1 | c2_second_stage | speak reask（L2_B3_REASK）回主等待 | speak reask（L3_REASK）回主等待 |
| 拒絕 gate NO | L2 resume、unclear 不動 | L3 resume、unclear 不動 | 同主迴圈 L2 | 同主迴圈 L3 |
| 想一下 | unclear=0；think++；≥3 退場 | unclear=0；think++；≥4 c2 | think++（**無 unclear 動作**）；≥3 退場 | think++（無 unclear）；≥4 c2 |
| 結帳 | unclear++；≥MAX 退場；否則 clarify | unclear=0；checkout_flow | clarify（**無 unclear++**） | checkout_flow（**無 unclear=0**） |
| 客服 yes | unclear=0；L2_ENTRY_PROMPT | unclear=0；L3_REASK | L2_ENTRY_PROMPT（無 unclear） | L3_REASK（無 unclear） |
| 客服 no | exit_a | exit_a | exit_a | exit_a |
| 商品 added | unclear=0；was_empty：think=0+do_action+TRANSITION | unclear=0；L3_REASK | do_action+TRANSITION（**think 不動**） | L3_REASK |
| 商品全 skip | unclear=0；reask 拼接 | 同 | reask 拼接（無 unclear） | 同 |
| 想買無商品 | 不動計數；VAGUE | 同 | 同 | 同 |
| 其他 unclear | unclear++；≥MAX 退場；clarify | unclear++；≥MAX final-confirm 流；clarify | clarify（**無計數**） | clarify（無計數） |

### 2-6. Quirk 保全清單（統一時最容易被「順手修掉」的三個現行行為——**全部保留**）

| # | Quirk | 保全機制 |
|---|---|---|
| Q1 | **L2 沉默鏈的 think 增量不回寫主迴圈**（原 `_dialog_think_silence_l2` 回 `None` 不回 int；L3 版回 int 回寫） | `silence_think_writeback`（L2 False）+ 主迴圈 snapshot/restore（§2-3） |
| Q2 | **inner 語境完全不碰 unclear_count**（含結帳分支——主迴圈 L2 結帳會 unclear++，inner L2 不會） | `on_checkout_inner` 與 `on_checkout_main` 分開成兩個 hook |
| Q3 | **inner 的商品 added 不 reset think_count**（B11 重置只發生在主迴圈：商品 was_empty 分支 + L2 silence 返回後 cart-transition 檢查） | inner 商品分支無 think 操作；主迴圈兩處 B11 原樣 |

依「發現 bug 直接修」常設指示評估過：三者皆**刻意設計或無害 quirk**（Q1/Q3 的計數差異被鏈內 limit 檢查兜住、Q2 是語境差異），非 bug，不修。
