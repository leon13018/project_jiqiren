## 2. 設計核心 + 行為規約

### 2-1. KeywordGroup（新檔 `myProgram/sales/keyword_group.py`）

```python
def contains_any(text: str, keywords) -> bool: ...        # 從 nlu.py 原樣搬入（邏輯一字不改）
def equals_strict_short(text: str, keywords) -> bool: ... # 從 nlu.py 原樣搬入（邏輯一字不改）

@dataclass(frozen=True)
class KeywordGroup:
    """keyword 雙集封裝：substring 比對集 + 嚴格相等集（防短詞 substring 誤命中）。"""
    substrings: tuple
    strict_short: tuple = ()

    def matches(self, text: str) -> bool:
        return contains_any(text, self.substrings) or equals_strict_short(text, self.strict_short)
```

- **檔案位置修正**（相對傘狀設計 §3）：類別放獨立 `sales/keyword_group.py` 而非 `constants/keywords.py`——constants 層規範「純資料無邏輯」（constants/CLAUDE.md），且原語放 keyword_group 可避免 keywords ↔ nlu 循環 import。傘狀設計 §3/§4-1 對應行隨本 wave 一併修正。
- **import 鏈**：`keyword_group.py` 只 import stdlib → `constants/keywords.py` import KeywordGroup 建實例 → `nlu.py` re-export 原語。無循環。
- **nlu.py 相容**：`contains_any` / `equals_strict_short` 本體搬走後，nlu.py 改 `from myProgram.sales.keyword_group import contains_any, equals_strict_short`（re-export）——所有 `from myProgram.sales.nlu import contains_any` 的既有 caller / 測試不受影響。

### 2-4. parse_quantity 合併（`nlu.py` + `product_parser.py`）

```python
def parse_quantity(text: str, default: int | None = 1) -> int | None:
```

- 行為規約（與兩個既有版本逐分支對齊）：
  1. 阿拉伯數字：第一個 >0 整數返回；全為 0 → 返 `0`（B16 顯式 0，**與 default 無關**）。
  2. 複合中文（十/百位）命中 → 返值。
  3. 單字中文映射命中 → 返值。
  4. 皆未命中 → **返 `default`**（既有 caller 不傳 = 1，行為不變；`default=None` = 原 `_parse_quantity_in_window` 語意）。
- `product_parser._parse_quantity_in_window` **刪除**，唯一呼叫點（`parse_products` 步驟 3）改 `parse_quantity(window, default=None)`；測試未 import 該私有函式（已 grep 驗證）。
- `product_parser.py` import 清理：`_parse_compound_chinese` 與 `CHINESE_DIGIT_MAP` 隨函式刪除而 unused → 移除；`KEYWORDS_ICED_TEA` / `KEYWORDS_SCRATCH` 疑似 unused → **先 grep 全 repo 確認無人經 product_parser 取用**才移除，有人用就保留並在回報註明。
