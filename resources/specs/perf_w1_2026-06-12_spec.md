### 2-2. F-1：KeywordGroup 預 lower＋nlu 呼叫點 KG 化

**KeywordGroup**（搬檔後的 `constants/keyword_group.py`）：

```python
@dataclass(frozen=True)
class KeywordGroup:
    substrings: tuple
    strict_short: tuple = ()

    def __post_init__(self):
        # 預先小寫化（perf_w1 F-1）：keyword 為 frozen 常數，lower 恆定；
        # 熱路徑 matches 不再每呼叫重算。frozen dataclass 用 object.__setattr__。
        object.__setattr__(self, "_substrings_lower",
                           tuple(kw.lower() for kw in self.substrings))
        object.__setattr__(self, "_strict_short_lower",
                           frozenset(kw.lower() for kw in self.strict_short))

    def matches(self, text: str) -> bool:
        text_lower = text.lower()
        return (any(kw in text_lower for kw in self._substrings_lower)
                or text_lower.strip() in self._strict_short_lower)
```

- 等價聲明：`text.strip().lower() == text.lower().strip()`（lower 映射不增減空白）；
  `in frozenset()` 與 `in []` 對空集皆 False；`eq`/`repr` 只看 dataclass 欄位，
  `test_kg_*_wired` 的 `.substrings`/`.strict_short` 斷言不受影響。
- 公開原語 `contains_any` / `equals_strict_short` **簽名與實作不動**（仍供 KG 外零散
  caller 用——本波後 nlu 內已無直呼，但原語是公開面，留待後續波評估）。

**nlu.py 呼叫點 KG 化**：新增模組級私有 KG 實例，classify_intent 內所有
`contains_any/equals_strict_short` 直呼改 `.matches`，**判定行序＝優先序逐行保留**：

| 新 KG 實例 | 來源集 | 取代的呼叫點（行為不變） |
|---|---|---|
| `_KG_REJECT_L3_STRICT` | `(_KEYWORDS_REJECT_L3_STRICT,)` | normal mode 拒絕判定 |
| `_KG_THINK` / `_KG_CHECKOUT` / `_KG_SERVICE` | 各自單集 | 通用優先序三判定 |
| `_KG_ICED_TEA` / `_KG_SCRATCH` | 商品雙集 | 商品判定 |
| `_KG_CONTINUE` / `_KG_EXIT` | l4_service 雙集 | l4_service 判定（行序 CONTINUE→EXIT→no/nope 不變） |
| `_KG_L4_ACK` | `(KEYWORDS_L4_ACK_OR_WAIT, KEYWORDS_L4_ACK_SHORT)` | l4 mode 兩個 if 合一（同回「等待安撫」，OR 語義等價） |
| `_KG_WANT_TO_BUY` | `(KEYWORDS_WANT_TO_BUY_VAGUE, KEYWORDS_WANT_TO_BUY_SHORT)` | 想買無商品兩個 if 合一（同回值，OR 等價） |
| `_KG_NO_NOPE` | `((), ("no", "nope"))` | l2/l4 拒絕＋l4_service 退出（兩處） |

`"停止" in text` 字面判定不動。`_KG_REJECT` 既有不動。
